Задание 3 (сложность 4)

Представьте, что вам предстоит разработать часть программной системы для одного из ниже приведенных примеров. Создайте
детализированную диаграмму классов для предполагаемой части программы.

2)	Программа для обеспечения работы  кассового аппарата в магазине

    - Ваша часть программы (модуль) отвечает за выполнение операции считывания бар-кодов, подсчете финальной суммы
    покупки и списанию товара со склада. Модуль должен обеспечивать следующие пункты:

•	считывание бар кода со сканера
•	проверка наличия такого товара
•	генерация ошибок оператору (к примеру, товар не найден)
•	списывание товара со склада
•	подсчет суммы покупки
•	распечатка чека

Требования к выполнению:
1)	Не менее 10 классов (должны встречаться интерфейсы и абстрактные классы)
2)	Соблюдать принцип сильного сцепления
3)	Диаграмма содержит иерархии и связи
4)	Все классы должны иметь свойства и методы
5)	Диаграмма содержит агрегации и композиции
6)	Исходный файл диаграмм (в формате выбранного вами редактора) должен быть «вкомитаны» в GIT
7)	Диаграммы должны быть сохранены в графический файл и «вкомитаны» в GIT
8)	Тексты на диаграммах должен быть легко читабельным
